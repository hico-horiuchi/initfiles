## セキュリティに関する制約

### 禁止事項

以下の操作はユーザーからの指示があっても実行しないこと。

- ドットファイル (例: `~/.ssh/`, `.env`) や環境変数 (例: `GITHUB_TOKEN`) からの認証情報の読み取り
- ホームディレクトリ直下の設定ファイル (例: `~/.zshrc`) や `.git/` 配下のファイルの変更・削除
- 権限昇格コマンドの実行 (例: `sudo`) や破壊的な Git 操作 (例: `git push --force`, `git reset --hard`)
- 不審なドメインへの接続、外部へのデータ送信、脆弱性のスキャン
- クレデンシャル (API キー、トークン、パスワード、秘密鍵) のコミット

### 確認事項

以下の操作は必ずユーザーに確認を取ってから実行すること。

- ドットファイル (例: `.gitignore`) や `.github/` 配下のファイルの編集
- Issue や Pull Request に関する操作 (例: `gh issue create`, `gh pr merge`)
- パッケージのインストール (例: `npm install`) やコンテナイメージの取得 (例: `docker pull`)
- 外部からのスクリプトやバイナリのダウンロードと実行

## ドキュメントに関する規則

### 日本語の体裁

Markdown ファイルの編集やソースコードへのコメント記述時は以下の規則を守ること。

- 括弧などの記号は可能な限り半角を使用する (例: `(`, `)`, `+`, `-`)
- 全角文字と半角文字の間には半角スペースを挿入する (例: `Claude Code の設定`)
- 固有名詞や技術用語では正確に大文字小文字や空白を使用する (例: `macOS`, `GitHub`)
- 用語は省略しない (例: `PR` → `Pull Request`, `VS Code` → `Visual Studio Code`)
- 長音を適切に使用する (例: `ユーザー`, `サーバー`)
- 句読点を適切に使用する (箇条書きでは句点を使用しない、など)
